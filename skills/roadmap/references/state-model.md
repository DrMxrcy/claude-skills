# Roadmap state model

## Source of truth
- **Plan files** (`.roadmap/plans/NNN-slug.md`) — progress (the checklist) + frontmatter.
- **`.roadmap/config.json`** — project, currentVersion, nextId, item registry, settings.
- **`ROADMAP.md`** — rendered view; the region between `<!-- roadmap:auto:start -->` and `<!-- roadmap:auto:end -->` is regenerated by `sync` (including the current-version line and per-version breakdown). Fully-done versions strictly below `currentVersion` collapse to a one-line summary unless `settings.collapseShipped` is `false`. Everything else (title, Idea Incubator) is free-form and never rewritten by scripts — `idea`/`remove` only append single bullets under the Idea Incubator heading.
- **Incubator location** — bullets live under the Idea Incubator heading in `ROADMAP.md` by default, or in the external file named by `settings.incubatorFile` (usually `.roadmap/IDEAS.md`, created by `tidy --externalize`, which leaves one linked bullet on the dashboard). `idea`/`promote`/`remove` resolve the location via `incubator_file()`; `tidy` analyzes both.
- **`.roadmap/notes/*.md`** — long-form idea write-ups created by `idea --body/--body-file`, linked from their incubator bullet. Not tracked in the registry.

## config.json
```json
{"project":"NAME","currentVersion":"0.0.1","nextId":1,
 "items":[{"id":1,"slug":"auth-setup","title":"Auth setup","type":"feature",
           "version":"0.0.1","file":"plans/001-auth-setup.md"}],
 "versionDates":{"0.0.1":"2026-07-09"},
 "settings":{"autoCommit":true,"gitTagOnRelease":false,"collapseShipped":true}}
```
- `versionDates` — stamped by `sync` the first time a version hits 100% (stable changelog/collapse dates).
- `settings.collapseShipped` — defaults to `true` when absent.
- `settings.incubatorFile` — when set (by `tidy --externalize`), the Idea Incubator lives in this file instead of `ROADMAP.md`.

## Plan frontmatter
- `id, title, type (feature|bug|refactor|chore), version, status, created`
- `status` is DERIVED by `sync` from checkboxes: planned (0 done) / active (some) / done (all).

## Invariants
- Never hand-edit the managed region of ROADMAP.md; run `sync`.
- All mutations go through `roadmap.py` (atomic writes; safe on partial failure).
- Checklist lines inside fenced code blocks are ignored by parsing/sync/import.
