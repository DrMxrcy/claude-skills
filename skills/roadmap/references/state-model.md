# Roadmap state model

## Source of truth
- **Plan files** (`.roadmap/plans/NNN-slug.md`) — progress (the checklist) + frontmatter.
- **`.roadmap/config.json`** — project, currentVersion, nextId, item registry, settings.
- **`ROADMAP.md`** — rendered view; the region between `<!-- roadmap:auto:start -->` and `<!-- roadmap:auto:end -->` is regenerated by `sync` (including the current-version line and per-version breakdown). Everything else (title, Idea Incubator) is free-form and never touched by scripts.

## config.json
```json
{"project":"NAME","currentVersion":"0.0.1","nextId":1,
 "items":[{"id":1,"slug":"auth-setup","title":"Auth setup","type":"feature",
           "version":"0.0.1","file":"plans/001-auth-setup.md"}],
 "settings":{"autoCommit":true,"gitTagOnRelease":false}}
```

## Plan frontmatter
- `id, title, type (feature|bug|refactor|chore), version, status, created`
- `status` is DERIVED by `sync` from checkboxes: planned (0 done) / active (some) / done (all).

## Invariants
- Never hand-edit the managed region of ROADMAP.md; run `sync`.
- All mutations go through `roadmap.py` (atomic writes; safe on partial failure).
- Checklist lines inside fenced code blocks are ignored by parsing/sync/import.
